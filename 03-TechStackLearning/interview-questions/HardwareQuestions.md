# 嵌入式硬件/板级调试常见基础概念与面试题

## 1. 什么是设备树（Device Tree）？为什么需要设备树？
- 设备树是一种描述硬件资源的数据结构，通常以 DTS/DTB 文件形式存在。
- 用于在系统启动时向内核传递硬件信息（如外设地址、中断、GPIO 配置等）。
- 使内核与硬件解耦，便于同一内核适配不同硬件平台。

## 2. 设备树中 GPIO 节点的常见属性有哪些？
- `gpios`：指定 GPIO 控制器和引脚编号。
- `gpio-controller`：声明该节点为 GPIO 控制器。
- `status`：使能/禁用该节点（"okay"/"disabled"）。
- `pinmux`：引脚复用配置。
- `bias-pull-up/down`：上拉/下拉配置。
- `output-high/low`：上电默认输出电平。

## 3. 如何通过设备树配置 GPIO 供电？
- 在设备树中为相关外设节点添加 `gpios` 属性，指定供电控制引脚。
- 可通过 `regulator` 子系统描述电源管理，绑定 GPIO 控制电源开关。
- 典型写法：  
  ```dts
  vcc-supply = <&vcc_reg>;
  enable-gpios = <&gpio1 5 GPIO_ACTIVE_HIGH>;
  ```

## 4. Linux 内核中 GPIO 的基本操作流程？
- 通过设备树或平台数据注册 GPIO。
- 驱动中通过 `devm_gpiod_get()`、`gpiod_direction_output()` 等 API 控制 GPIO。
- 用户空间可通过 sysfs、/dev/gpiochipX 或 libgpiod 操作 GPIO。

## 5. 什么是引脚复用（Pinmux）？为什么需要？
- 一根物理引脚可复用为多种功能（如 GPIO、UART、I2C、SPI）。
- 通过设备树或寄存器配置选择具体功能，提升引脚利用率。

## 6. 常见的电源管理方式有哪些？
- 软件控制（GPIO 控制 LDO、DCDC 使能）。
- 硬件自动控制（PMIC、RTC 定时唤醒）。
- 动态电压频率调整（DVFS）。
- 休眠唤醒机制（Wakeup Source）。

## 7. 板级调试常用的接口和工具有哪些？
- 串口（UART）：打印 log，调试 bootloader、内核。
- JTAG/SWD：硬件级调试、单步跟踪、烧录。
- I2C/SPI 总线分析仪：抓取通信数据。
- 示波器/逻辑分析仪：观测信号波形。
- 多用电表：测量电压、电流。

## 8. 如何定位 GPIO 不工作的常见原因？
- 检查设备树配置是否正确（引脚编号、复用、上下拉）。
- 硬件连线是否正确，供电是否正常。
- 驱动是否正确加载，内核 log 是否有报错。
- 用示波器/万用表实际测量电平变化。

## 9. 设备树的 overlay 机制是什么？
- overlay 允许在运行时动态修改设备树，适用于可插拔模块、扩展板等场景。
- 常用于开发板外设扩展、热插拔设备支持。

## 10. 什么是 regulator 子系统？如何与 GPIO 配合实现电源控制？
- regulator 子系统用于统一管理板上各类电源（LDO、DCDC）。
- 可通过设备树将 regulator 与 GPIO 绑定，实现软件控制电源开关。
- 便于电源时序管理、低功耗控制。

## 11. UART、I2C、SPI 的基本原理和区别是什么？
- UART（通用异步收发器）：点对点异步串口通信，无时钟线，常用于串口调试和低速外设。优点是简单、成本低，缺点是只能点对点、速率较低。
- I2C（Inter-Integrated Circuit）：双线（SCL/SDA）同步串行总线，支持多主多从，适合短距离、低速外设（如传感器、EEPROM）。优点是节省引脚、支持挂载多个设备，缺点是速率有限、易受干扰。
- SPI（Serial Peripheral Interface）：四线（SCLK/MOSI/MISO/CS）同步串行总线，主从结构，速率高，适合高速外设（如 Flash、LCD）。优点是速度快、协议简单，缺点是需要更多引脚、通常只支持单主多从。

## 12. UART 通信常见参数有哪些？如何配置？
- 波特率（Baudrate）、数据位（Data bits）、停止位（Stop bits）、校验位（Parity）、流控（Flow control）。
- 配置时需保证通信双方参数一致，常见配置如 115200 8N1（8 数据位，无校验，1 停止位）。

## 13. I2C 地址冲突如何解决？I2C 总线挂载多个设备时注意什么？
- I2C 设备通过 7 位或 10 位地址区分，若地址冲突可通过硬件引脚配置或更换器件解决。
- 多设备时注意上拉电阻配置、总线长度、地址唯一性，避免信号干扰。

## 14. SPI 的主从模式和片选信号（CS）作用是什么？
- SPI 由主设备控制时钟和片选，片选（CS）用于选择具体的从设备。
- 多个从设备时，每个从设备需独立的 CS 信号，主设备通过拉低对应 CS 与目标设备通信。

## 15. 如何在设备树中配置 I2C/SPI 外设？
- 需在设备树中声明 I2C/SPI 控制器节点及其下挂的外设节点，指定地址、兼容性字符串、所用引脚等。
- 例如：
  ```dts
  i2c@40005400 {
      status = "okay";
      sensor@48 {
          compatible = "xxx,sensor";
          reg = <0x48>;
      };
  };
  ```

## 16. UART/I2C/SPI 调试常用方法有哪些？
- UART：串口工具（minicom、SecureCRT）、示波器观测波形。
- I2C：i2cdetect/i2cget/i2cset 工具，逻辑分析仪抓包。
- SPI：spidev 测试、逻辑分析仪抓包，驱动 log 分析。

## 17. I2C/SPI 驱动开发常见流程？
- 注册总线控制器驱动，编写外设驱动（probe/remove）。
- 通过设备树匹配设备，驱动中调用 i2c_transfer/spi_sync 等 API 进行数据收发。
- 处理中断、DMA、错误检测等。

## 18. UART 波特率不匹配会出现什么现象？如何排查？
- 波特率不一致会导致乱码、丢包、通信失败。
- 排查方法：确认双方波特率设置一致，检查硬件连线、终端配置，使用示波器测量实际波形。

## 19. I2C 总线挂载过多设备会出现什么问题？如何优化？
- 设备过多会导致总线负载大、信号上升沿变慢、通信不稳定。
- 优化方法：合理选择上拉电阻、缩短总线长度、分段挂载、降低速率。

## 20. SPI 支持全双工还是半双工？如何实现多设备通信？
- SPI 支持全双工通信（MOSI/MISO 可同时收发）。
- 多设备通信通过独立的 CS 片选线实现，主设备轮流与各从设备通信

## 21. UART 通信有哪些常见异常？如何排查？
- 常见异常：乱码、丢包、通信中断、噪声干扰。
- 排查方法：检查波特率、数据位、停止位、校验位设置是否一致；检查硬件连线和接地；用示波器观测波形；排除电磁干扰。

## 22. I2C 的时序要求和典型问题有哪些？
- 时序要求：SCL/SDA 上升/下降沿、起始/停止条件、数据保持时间等需满足协议规范。
- 典型问题：时序不符导致通信失败、总线被拉低（死锁）、重复起始条件处理异常。
- 解决：合理配置上拉电阻，保证信号完整性，分析波形排查。

## 23. SPI 的时序参数有哪些？如何配置？
- 主要参数：时钟极性（CPOL）、时钟相位（CPHA）、时钟频率、数据位宽。
- 配置需保证主从设备参数一致，常见模式有四种（CPOL/CPHA 组合）。

## 24. CAN 总线是什么？有哪些典型应用？
- CAN（Controller Area Network）是一种多主机、差分信号的串行通信总线，抗干扰能力强。
- 典型应用：汽车电子、工业自动化、医疗设备等。
- 特点：支持多节点、优先级仲裁、错误检测机制。

## 25. CAN 总线的帧结构和通信机制？
- 帧结构包括：起始位、仲裁段、控制段、数据段、CRC 校验、应答段、结束位。
- 通信机制：采用优先级仲裁，支持标准帧（11位ID）和扩展帧（29位ID）。

## 26. RS485 总线是什么？与 UART 有什么区别？
- RS485 是一种差分信号的半双工串行通信标准，支持多点总线拓扑，抗干扰能力强，适合远距离通信。
- 与 UART 区别：UART 通常为点对点单端通信，RS485 可多点、长距离、差分传输。

## 27. QSPI（Quad SPI）是什么？与普通 SPI 有何区别？
- QSPI 是 SPI 的扩展，支持 4 线数据并行传输（IO0~IO3），大幅提升数据速率。
- 主要用于高速 Flash 存储器、显示屏等场景。
- 区别：普通 SPI 单线收发，QSPI 可四线并发，速度更快。

## 28. CAN、RS485、SPI、I2C 适用场景对比？
- CAN：多节点、抗干扰、实时性高，适合汽车、工业控制。
- RS485：远距离、半双工、多点通信，适合楼宇、工业自动化。
- SPI：高速、短距离、主从结构，适合芯片间通信（如 Flash、LCD）。
- I2C：低速、短距离、节省引脚，适合传感器、EEPROM 等。

## 29. 如何判断 SPI/I2C/CAN 通信异常？常用调试手段有哪些？
- SPI/I2C：用逻辑分析仪抓包，检查波形、时序、ACK 信号。
- CAN：用 CAN 分析仪抓包，检查帧格式、错误计数、仲裁冲突。
- 检查驱动 log、硬件连线、终端电阻配置。

## 30. QSPI 在嵌入式系统中的典型应用？
- 常用于高速 NOR Flash 存储器，提升固件加载和数据读写速度。
- 也用于高分辨率显示屏、音视频数据传输等对带宽有要求的场景。
