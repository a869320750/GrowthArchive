## 1.1 将序列分解为单独变量
### 问题
定长列表/元组，一次性拆解赋值给多个变量。
### 工具
序列解包 `a,b,c = seq`
### 要点
1. 可迭代对象（list/tuple/字符串/生成器）都能解包；
2. 嵌套序列支持多层解包；
3. 元素数量必须和变量数量严格匹配，否则抛异常。

## 1.2 从任意长度可迭代对象解压元素
### 问题
序列长度不确定，只想取头部/尾部，中间批量丢弃或保存。
### 工具
星号解包 `head, *middle, tail = seq`
### 要点
1. `*变量` 接收任意多个元素，自动包装成列表；
2. 丢弃无用数据用 `_` 占位；
3. 适合文件行、日志、变长接口数据解析。

## 1.3 保留最后 N 个元素
### 问题
流式读取数据，只缓存最近几条历史记录，控制内存。
### 工具
`from collections import deque(maxlen=N)`
### 要点
1. 定长双端队列，满了自动淘汰最早元素；
2. `append()` 尾部追加，`appendleft()` 头部插入；
3. 两端增删性能远优于 list。

## 1.4 查找最大/最小的 N 个元素
### 问题
海量序列取 TopN，不完整全排序节省性能。
### 工具
`heapq.nlargest(n, iterable, key)` / `heapq.nsmallest()`
### 要点
1. 底层最小堆/最大堆实现，数据量大远快于 `sorted`；
2. 支持 `key` 自定义比较规则；
3. N 接近序列总长时，直接用 `sorted` 更快。

## 1.5 实现优先级队列
### 问题
元素带权重，随时插入并持续弹出优先级最高项。
### 工具
`heapq` 模块 + `(优先级, 计数, 对象)` 三元组堆
### 要点
1. list 原地堆化 `heapify`；`heappush`/`heappop`；
2. 增加自增计数防止同优先级对象比较报错；
3. 工业级简易调度队列标准实现。

## 1.6 一键多值字典（multidict）
### 问题
一个 key 对应一组值，不用手动判断 key 是否存在再初始化空容器。
### 工具
`from collections import defaultdict`
- `defaultdict(list)`：有序、允许重复，`append()`
- `defaultdict(set)`：自动去重，`add()`
### 要点
1. 缺失 key 自动调用工厂函数生成空容器；
2. 替代 `dict.setdefault`，性能、可读性更优。

## 1.7 字典保持插入顺序
### 问题
遍历、序列化 JSON 时严格控制字典输出顺序。
### 工具
`from collections import OrderedDict`
### 要点
1. Python3.7+ 原生 dict 保插入顺序，但 OrderedDict 独有特性：
   - `move_to_end(key)`：移动键到首尾
   - `popitem(last=False)`：弹出最早插入元素
   - `==` 相等判断区分顺序（普通 dict 忽略顺序）
2. 适合 LRU 缓存、严格有序配置文件。

## 1.8 字典执行计算（最大、最小、排序）
### 问题
对字典的键值做数值运算、按值排序。
### 工具
`zip(d.values(), d.keys())` 生成 `(值, 键)` 元组
### 要点
1. `min/max/sorted` 按元组第一个元素（数值）比较；
2. zip 返回一次性迭代器，重复使用要转 list；
3. 可直接传入 `heapq` 取价格最高/最低股票。

## 1.9 找出两个字典的交集、差集、相同键值
### 问题
对比两个字典，提取共有键、独有键、相同键值对。
### 工具
`dict.keys()` / `dict.items()` 视图对象（支持集合运算符）
### 集合运算符
- `&` 交集、`|` 并集、`-` 差集、`^` 对称差集
### 要点
1. keys 视图天然支持集合运算，无需转 set；
2. values 不支持集合运算（值可重复）。

## 1.10 去重序列且保留原有顺序
### 问题
普通 `set()` 去重打乱顺序，需要维持原始先后。
### 工具
生成器 `yield` + `set` 判重缓存
### 要点
1. 生成器惰性迭代，大序列节省内存；
2. 支持自定义 `key` 函数：复杂对象按指定字段去重；
3. 仅可哈希元素可用（数字、字符串、元组）。

## 1.11 命名切片
### 问题
长字符串固定偏移截取，硬编码数字可读性差、难维护。
### 工具
`slice(start, stop)` 切片对象
### 要点
1. 统一命名切片常量，一处修改全局生效；
2. 老式固定宽度文本、日志解析专用。

## 1.12 统计序列元素出现次数，找出频次最高项
### 问题
批量计数，快速取 TopN 高频元素。
### 工具
`from collections import Counter`
### 要点
1. dict 子类，构造时一次性完成全量统计；
2. `.most_common(n)` 直接返回频次前N；
3. 支持 `update()` 增量追加计数、Counter 加减运算。

## 1.13 字典列表按字典字段排序
### 问题
列表内全是字典，根据某个 key 批量排序。
### 工具
`sorted()` + `operator.itemgetter(字段名)`
### 要点
1. itemgetter 底层C实现，性能优于 lambda；
2. 支持多字段排序 `itemgetter('lname','fname')`；
3. 搭配 `list.sort()` 原地排序。

## 1.14 自定义对象排序（无原生比较）
### 问题
自定义类未实现 `__lt__`，无法直接 `<` 比较，需要按属性排序。
### 工具
1. `lambda u: u.user_id`
2. `operator.attrgetter('属性名')`（优先推荐）
### 要点
attrgetter 多属性排序写法简洁、速度更快。

## 1.15 根据字段对记录分组
### 问题
一组字典/对象，按同一个字段聚合分组。
### 工具
`itertools.groupby()` + `defaultdict(list)`
### 要点
1. groupby 仅对**连续相同 key** 分组，使用前必须先排序；
2. defaultdict 存放分组结果，一键聚合。

## 1.16 过滤序列元素
### 问题
按条件筛选数据，分内存充足/内存敏感两种场景。
### 工具
1. 列表推导式 `[x for x in seq if cond]`（生成完整列表）
2. 生成器表达式 `(x for x in seq if cond)`（惰性迭代，省内存）
### 补充：复杂过滤规则用 `itertools.filterfalse`。

## 1.17 从字典中提取子集（筛选键）
### 问题
新建字典，只保留指定键、剔除部分键。
### 工具
**字典推导式** `{k: d[k] for k in d.keys() - {'a','b'}}`
### 要点
利用 dict_keys 集合差集快速过滤键，一行替代循环判断。

## 1.18 将名称映射到序列元素（命名元组）
### 问题
列表/元组靠下标取值可读性差，想用字段名访问。
### 工具
`from collections import namedtuple`
### 要点
1. `Stock = namedtuple("Stock", ["name","shares","price"])`；
2. `*rec` 序列解包批量赋值；`**dict` 字典解包构造；
3. 相比普通类更轻量，适合小型数据记录。

## 1.19 同时转换、计算序列数据
### 问题
遍历序列同时完成数据转换与累加，不用多次循环。
### 工具
生成器表达式放入聚合函数 `sum/max/min`
```python
total = sum(int(rec[s]) * float(rec[p]) for rec in records)
```
### 要点
单次遍历完成取值、类型转换、求和，减少循环次数。

## 1.20 多个字典逻辑合并查询（不创建新字典）
### 问题
多个字典分层查询：优先查第一个，无则向后查找，不拷贝合并。
### 工具
`from collections import ChainMap(a,b,c)`
### 要点
1. 不复制原字典，修改原数据 ChainMap 同步生效；
2. 对比 `dict.update()`：后者生成全新字典，内存开销大；
3. 写操作仅作用在第一个字典，只读查询逐层向后匹配。

# 第一章核心工具总表（速查）
## 1. collections 五大高频容器
1. `deque`：定长队列、两端高效增删
2. `defaultdict`：一键多值自动初始化字典
3. `OrderedDict`：有序字典、LRU缓存专用
4. `Counter`：频次统计、TopN
5. `namedtuple`：带字段名的轻量元组
6. `ChainMap`：多字典分层视图

## 2. 底层算法工具
- `heapq`：堆、TopN、优先级队列

## 3. operator 取值工具（排序专用，性能优于lambda）
- `itemgetter`：字典key、序列下标取值
- `attrgetter`：自定义对象属性取值

## 4. 语法糖/内置表达式
1. 序列解包 `a,*b,c = seq`
2. 三元表达式 `val = x if cond else y`
3. 三大推导式：列表/集合/字典推导
4. 生成器 `yield`、生成器表达式 `(x for x in ...)`
5. `slice()` 命名切片
6. zip 配对迭代

## 5. 通用内置函数
- `sorted()`：全局排序，返回新列表
- `list.sort()`：列表原地排序，无返回值
- `min/max` 配合zip实现字典数值运算

# 本章设计思想总结（你之前关心的语言取舍）
1. Python 大量提供**专精小工具**而非统一万能接口：堆、有序字典、计数、多值字典分开实现，针对性优化性能；
2. 鸭子类型贯穿全章：迭代、比较、哈希均靠魔术方法隐式实现，无强制接口约束；
3. 两套内存方案全覆盖：需要完整结果用列表/普通字典；海量流式数据用生成器、deque、ChainMap惰性加载；
4. 所有排序、过滤、集合操作统一接口设计，`key` 参数规则完全互通，降低记忆成本。
